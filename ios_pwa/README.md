# Store Checklist iPhone PWA

Эта папка содержит отдельную веб-версию приложения для iPhone (PWA).
Android-код (`app/`) и сервер (`local_server/`) не изменяются.

## Что уже перенесено

- Роли: `Управление`, `Super user`, `Пользователь`.
- Настройки:
  - тема (`система / светлая / темная`);
  - режим прохождения (`скрывать при нажатии / маркер`);
  - `Server URL`, `Read token`, `Write token`.
- Локальная история на устройстве (через `localStorage`):
  - списки, товары, прогресс прохождения.
- Синхронизация с текущим сервером:
  - обычный режим: импорт только отсутствующих списков с сервера;
  - `Super user`: полная замена серверной базы локальной.
- PWA-режим:
  - `manifest`,
  - `service worker`,
  - установка ярлыка на экран Домой.

## Структура папки

- `index.html`, `styles.css`, `app.js` - само веб-приложение.
- `manifest.webmanifest` - конфигурация PWA.
- `sw.js` - offline-кэш оболочки приложения.
- `icons/` - иконки для установки на iPhone.
- `serve_local.ps1` - локальный запуск статики.
- `package_pwa.ps1` - упаковка папки в zip для переноса/деплоя.

## Быстрый запуск локально (на ПК)

1. Перейдите в папку PWA:

```powershell
cd ios_pwa
```

2. Запустите статический сервер:

```powershell
.\serve_local.ps1
```

3. Откройте в браузере:

```text
http://127.0.0.1:8787
```

## Как связать с вашим API сервером

В настройках PWA укажите:

- `Server URL`: например `https://your-domain.com/api/`
- `Read token`: токен чтения (если включен на сервере)
- `Write token`: токен записи (для `Super user`)

Важно:

- URL должен заканчиваться на `/api/`.
- Для внешнего адреса используйте `https://`.
- Для локальной сети допустим `http://192.168.x.x:8080/api/`.

## Как перенести на iPhone и установить

Ниже рабочая схема для реального использования.

1. Опубликуйте папку `ios_pwa` на HTTPS-хостинге (Nginx, Caddy, VPS, статический хостинг).
2. Проверьте, что сайт открывается по адресу вида:
   - `https://your-domain.com/` (или отдельный путь, например `/pwa/`).
3. На iPhone откройте именно в `Safari`.
4. Нажмите `Поделиться` -> `Добавить на экран Домой`.
5. Если iOS показывает переключатель `Open as Web App`, включите его.
6. Запустите иконку с экрана Домой.
7. В PWA откройте `Настройки`, задайте `Server URL` и токены, нажмите `Сохранить подключение`.

## Сборка/передача как архив

Скрипт создаст архив `store-checklist-ios-pwa.zip` рядом с папкой `ios_pwa`:

```powershell
cd ios_pwa
.\package_pwa.ps1
```

Архив можно:

- передать на сервер и распаковать в корень статики,
- либо загрузить в панель хостинга как статический сайт.

## Проверка перед релизом

1. Откройте PWA на iPhone через HTTPS.
2. Создайте локальный список, закройте Safari, откройте снова - список должен сохраниться.
3. Проверьте `Синхронизацию` в обычном режиме (импорт с сервера).
4. Проверьте `Super user` только на тестовом сервере, потому что он переписывает серверную базу полностью.
5. Проверьте работу без сети: запущенный ранее PWA должен открываться, а локальные списки должны быть доступны.
