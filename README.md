# Store Checklist (Android)

Приложение чек-листов для магазина с 2 ролями:
- Администратор: управляет списками/товарами и запускает синхронизацию.
- Пользователь: проходит выбранный список в двух режимах (скрытие или маркер).

## Главная идея для вашего сценария

Вместо внешнего сервера используется текущий ПК:
- на ПК запускается локальный HTTP-сервер одной командой в терминале;
- Android-устройства подключаются к нему по локальной сети (Wi-Fi);
- в приложении можно задать URL сервера (без пересборки APK).

Это позволяет собрать один APK и установить его на несколько сторонних устройств.

## Что уже реализовано

- Локальное хранение списков на устройстве (`Room`).
- Полная работа офлайн, даже если сервер недоступен.
- Подтягивание новых списков с сервера:
  - вручную кнопкой `Sync` в окне администратора;
  - фоново через `WorkManager`.
- Локальный сервер в репозитории (`local_server/`):
  - `GET /health`
  - `GET /api/checklists`
- CLI для “администратора сервера” в терминале (`local_server/manage.py`).
- В приложении добавлено поле `Server URL` для настройки адреса сервера на устройстве.

## Структура проекта

- `app/` - Android-приложение (Compose + Room + Retrofit + WorkManager).
- `local_server/` - локальный сервер и утилита управления данными.
- `run_local_server.ps1` - быстрый запуск сервера на Windows.

## 1) Как запускать локальный сервер с терминала

Требования:
- Python 3.10+ установлен на ПК.
- ПК и телефоны находятся в одной Wi-Fi сети.

Запуск из корня проекта:

```powershell
.\run_local_server.ps1
```

Альтернатива:

```powershell
py -3 .\local_server\server.py --host 0.0.0.0 --port 8080
```

Проверка с ПК:

```powershell
curl http://127.0.0.1:8080/health
curl http://127.0.0.1:8080/api/checklists
```

Сервер читает данные из:
- `local_server/data/checklists.json`

## 2) Как администратору сервера добавлять/менять списки через терминал

Показать все списки:

```powershell
py -3 .\local_server\manage.py list
```

Добавить новый список:

```powershell
py -3 .\local_server\manage.py add-list --title "Склад A" --item "Вода" --item "Сок" --item "Чай"
```

Переименовать список:

```powershell
py -3 .\local_server\manage.py rename-list --id <CHECKLIST_ID> --title "Склад A - утро"
```

Заменить товары списка:

```powershell
py -3 .\local_server\manage.py replace-items --id <CHECKLIST_ID> --item "Батон" --item "Молоко"
```

Удалить список:

```powershell
py -3 .\local_server\manage.py delete-list --id <CHECKLIST_ID>
```

## 3) Как собрать один APK для установки на сторонние устройства

Вариант через Android Studio (рекомендуется):

1. Откройте проект в Android Studio.
2. Дождитесь Gradle Sync.
3. Для тестов:
   - `Build` -> `Build Bundle(s) / APK(s)` -> `Build APK(s)`.
4. Для распространения:
   - `Build` -> `Generate Signed Bundle / APK` -> `APK`.
   - Создайте/выберите keystore.
   - Соберите `release` APK.

Вы получите один `.apk`, который можно установить на несколько телефонов.

Варианты установки APK:
- Через USB + ADB:
  - `adb install -r <path_to_apk>`
- Через файл:
  - отправьте APK (Telegram/почта/облако),
  - откройте файл на телефоне и разрешите установку из неизвестного источника.

## 4) Настройка URL сервера в приложении на телефоне

На ПК узнайте локальный IP (пример):

```powershell
ipconfig
```

Нужен IPv4 адрес активного адаптера, например `192.168.1.50`.

Тогда в приложении на телефоне в окне администратора в поле `Server URL` укажите:

```text
http://192.168.1.50:8080/api/
```

Нажмите `Save server URL`, потом `Sync`.

## 5) Порядок первого тестирования на реальном телефоне

1. На ПК запустите сервер:
   - `.\run_local_server.ps1`
2. На ПК проверьте, что сервер отвечает:
   - `curl http://127.0.0.1:8080/health`
3. На ПК добавьте тестовый список:
   - `py -3 .\local_server\manage.py add-list --title "Тест 1" --item "Хлеб" --item "Молоко"`
4. Установите APK на телефон.
5. Откройте приложение -> выберите `Admin`.
6. Введите `Server URL` вида `http://<IP_ПК>:8080/api/` и нажмите `Save server URL`.
7. Нажмите `Sync` и убедитесь, что список пришёл с сервера.
8. Перейдите в окно `User` -> откройте этот список.
9. Протестируйте режим 1 (`hide on tap`):
   - при нажатии товар исчезает из отображения.
10. Протестируйте режим 2 (`marker`):
   - при нажатии возле товара включается/выключается маркер.
11. Нажмите `reset progress` и проверьте, что список снова полный.
12. Выключите сервер на ПК (Ctrl+C), повторно откройте приложение:
   - локальные списки должны оставаться доступными (офлайн-работа).

## Важные замечания

- Телефон и ПК должны быть в одной сети, а порт `8080` не должен блокироваться firewall.
- В приложении включён `usesCleartextTraffic=true`, поэтому HTTP в локальной сети разрешён.
- Если IP ПК изменился, на устройстве нужно обновить `Server URL`.
